### 2. Bug Fixes

#### TSV Parsing Bug (Critical) - THE ACTUAL ROOT CAUSE
**Problem:** Unbalanced quotes in TSV explanations caused parser to enter "inQuotes" mode and swallow all subsequent rows. "Profit Loss Discount" was missing 85 questions.

**Root Cause:** Line 231 of the TSV file had 5 quotes (odd number), including a stray `18 less"` that broke parsing.

**Fix:** Modified `parseDelimited()` in `questionService.ts` to use simple split for TSV files:
```typescript
if (delim === '\t') {
  return content
    .split(/\r?\n/)
    .filter(line => line.trim().length > 0)
    .map(line => line.split('\t').map(field => field.trim()));
}
```

**Note on Database Constraints:** Earlier in the session, I incorrectly suspected a database constraint (`level <= 5`) was blocking level 6 uploads. This was WRONG - I was looking at a stale migration file that doesn't reflect the actual database. The database accepts levels 1-6+ with no issues. The ONLY problem was the TSV parsing bug.

#### Performance Lag Fix (Question Transition Speed)
**Problem:** User reported significant lag between questions - celebrations, sounds, animations happening too frequently.

**Root Cause:** Multiple overlapping delays:
1. Staggered option animations (0.1s per option = 0.4s total)
2. Fun elements showing 15% of correct answers
3. Character feedback at streak >= 4
4. Confetti at streak >= 3
5. Milestone animation lasting 2500ms

**Fix - Multiple files updated:**

| File | Before | After |
|------|--------|-------|
| `QuizCard.tsx` | Staggered animations 0.1s delay per option | Near-instant opacity transitions (0.1s total) |
| `feedbackService.ts` | Fun elements 15% chance | 5% chance |
| `feedbackService.ts` | Character at streak >= 4 | Streak >= 7 |
| `feedbackService.ts` | Struggle at 3 wrong | 4 wrong |
| `Index.tsx` | Confetti at streak >= 3 | Streak >= 5 |
| `MilestoneAnimation.tsx` | 2500ms duration | 1500ms duration |
| `useSoundEffects.ts` | Streak sounds at 3+/5+ | 5+/10+ |

**Result:** Question-to-question flow is now much snappier with minimal interruptions.
